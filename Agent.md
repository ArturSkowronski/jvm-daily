# Agent Instructions

When working on this project, ALWAYS follow these rules:

1. **Minimal implementation + unit tests** — implement the smallest working version, then write tests for it.
2. **Always build** — run `./gradlew build` after changes to verify everything works.
3. **Work on branches** — never commit to `main` directly. Create feature branches.
4. **Document findings** — save interesting discoveries, problems, and solutions to `Findings.md`.
5. **Push and PR** — when done, push the branch to GitHub and create a Pull Request.
